GET /api/onboarding/attributes?profileId=xxx
  Response: {
    attributes: Array<{
      attributeId: string;
      displayName: string;

      valueType: "BOOL" | "ENUM" | "REPEATED_ENUM" | "URL";

  Request: {

      valueType: string;
      values?: unknown[];
      repeatedEnumValue?: { setValues: string[], unsetValues: string[] };
      uriValues?: Array<{ uri: string }>;
    }>
  }
  Response: { success: boolean, error?: string }

GET /api/onboarding/settings?profileId=xxx
  Response: { postFrequency: number }

PATCH /api/onboarding/settings
  Request: { profileId: string, postFrequency: number }
  Response: { success: boolean }

GET /api/onboarding/summary?profileId=xxx
  Response: {
    steps: Array<{ name: string; status: "complete" | "skipped" | "pending"; detail?: string }>,

  }

Create a "use client" component: `AttributesStep({ profileId, onComplete }: { profileId: string; onComplete: () => Promise<void> })`.

**Types (local to file):**
```typescript
interface GBPAttribute {
  attributeId: string;
  displayName: string;
  groupDisplayName: string;
  valueType: "BOOL" | "ENUM" | "REPEATED_ENUM" | "URL";
  values: unknown[];
  valueMetadata?: Array<{ value: string; displayName: string }>;
}

// Local state for tracking user's selections per attribute
interface AttributeState {
  attributeId: string;
  displayName: string;
  groupDisplayName: string;
  valueType: "BOOL" | "ENUM" | "REPEATED_ENUM" | "URL";
  boolValue?: boolean;
  enumValue?: string;
  repeatedEnumValues?: string[];
  urlValue?: string;
  valueMetadata?: Array<{ value: string; displayName: string }>;
}
```

**State:**
- `attributes: AttributeState[]` -- parsed from API response with current values pre-filled
- `loading: boolean` -- initial fetch
- `pushing: boolean` -- push in progress
- `pushSuccess: boolean` -- shows success banner
- `pushError: string | null` -- shows error banner
- `collapsedGroups: Set<string>` -- tracks which groups are collapsed (all start expanded, i.e. empty set)
- `noAttributes: boolean` -- true if API returned empty array

**On mount (useEffect):**
1. Fetch GET /api/onboarding/attributes?profileId={profileId}
2. If response.attributes is empty array:
   - Set `noAttributes: true`, show "No attributes available for this business category" message
   - After 1500ms delay, auto-call `onComplete()` to skip step
   - Return early (do not render controls)
3. Parse response.attributes into AttributeState[]:
   - For BOOL: extract `boolValue` from `values[0]` (true/false), default false
   - For ENUM: extract `enumValue` from `values[0]` (string), default empty string
   - For REPEATED_ENUM: extract `repeatedEnumValues` from `values` array (string[]), default empty array
   - For URL: extract `urlValue` from `values[0]` (string or object with uri property), default empty string
4. Set `loading: false`

**Grouping and display:**
- Group attributes by `groupDisplayName` using a Map or reduce
- Render each group as a collapsible section:
  - Group header: clickable div with group name (font-medium text-gray-900) + ChevronDown/ChevronRight icon
  - On click: toggle group in `collapsedGroups` set
  - All groups start expanded (collapsedGroups starts as empty Set)
- Within each group, render attributes in a list with appropriate controls:

**Controls per valueType:**
- **BOOL**: Toggle switch (a styled checkbox input with `peer` Tailwind pattern or a simple button that toggles). Show attribute displayName next to toggle. On toggle: update `boolValue` in state.
  - Implementation: Use a button with rounded-full styling that slides a circle left/right. When on: bg-blue-600. When off: bg-gray-200. Size: w-10 h-6 with a w-4 h-4 circle.
- **ENUM**: Radio button group. Show attribute displayName as label. Below it, render radio buttons from `valueMetadata` array. Each radio shows the `displayName` from metadata. On select: update `enumValue` in state.
- **REPEATED_ENUM**: Checkbox group. Show attribute displayName as label. Below it, render checkboxes from `valueMetadata` array. On toggle: add/remove from `repeatedEnumValues` in state.
- **URL**: Text input. Show attribute displayName as label. Input with type="url", placeholder="https://...", value bound to `urlValue`. On change: update `urlValue` in state.

**Each attribute row styling:**
- Container: py-3 border-b border-gray-100 last:border-b-0
- Label: text-sm text-gray-700
- Controls: standard form element sizing

**"Push to Google" button (bottom of step):**
- Prominent: bg-green-600 text-white rounded-md font-medium px-6 py-2.5 w-full
- Disabled while pushing
- Shows Loader2 spinner + "Pushing to Google..." text while pushing
- On click:
  1. Set `pushing: true`, clear `pushError`
  2. Build payload from current attribute states:
     - BOOL: `{ attributeId, valueType: "BOOL", values: [boolValue] }`
     - ENUM: `{ attributeId, valueType: "ENUM", values: [enumValue] }` (skip if enumValue is empty)
     - REPEATED_ENUM: `{ attributeId, valueType: "REPEATED_ENUM", repeatedEnumValue: { setValues: repeatedEnumValues, unsetValues: [] } }`
     - URL: `{ attributeId, valueType: "URL", uriValues: [{ uri: urlValue }] }` (skip if urlValue is empty)
  3. POST /api/onboarding/attributes/push with { profileId, attributes: payload }
  4. If success: set `pushSuccess: true`, after 2500ms call `onComplete()`
  5. If failure: set `pushError: response.error || "Failed to push attributes to Google"`
  6. Set `pushing: false`

**Success banner:** bg-green-50 border border-green-200 rounded-lg p-4, CheckCircle2 icon + "Attributes successfully pushed to Google Business Profile" + "Advancing to next step..." subtext. Same pattern as description-step.tsx.

**Error banner:** bg-red-50 border border-red-200 rounded-lg p-4, AlertCircle icon + error message + "Retry" button (bg-red-600 text-white) that retries the push.

**"Skip for Now" link:**
- text-gray-500 underline text-sm, below the push button
- On click: call `onComplete()` directly (no save needed, attributes are GBP-only)

**Loading state:**
- Centered Loader2 spinner with animate-spin, same as description-step.tsx loading state

**No attributes state:**
- Centered SlidersHorizontal icon (text-gray-300 w-12 h-12) + "No attributes available for this business category" message + "Auto-advancing..." subtext

**Icons needed:** SlidersHorizontal, Loader2, CheckCircle2, AlertCircle, ChevronDown, ChevronRight from lucide-react
