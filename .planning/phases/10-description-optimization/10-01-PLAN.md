**1. Create src/lib/description-generator.ts:**

Define a Zod schema for structured AI output:
```typescript
const DescriptionSchema = z.object({
  description: z.string().max(750),
});
```

Export `generateDescription(profile: { name: string; category: string | null; address: string | null; keywords: string[]; cities: string[] }): Promise<string>`.

Implementation:
- Use `anthropic.messages.parse()` with `zodOutputFormat(DescriptionSchema)` — same pattern as keyword-generator.ts
- Model: "claude-sonnet-4-5-20250929"
- System prompt should instruct Claude to write a compelling, SEO-optimized Google Business Profile description. Key instructions:
  - Maximum 750 characters (HARD LIMIT — the output must be under 750 chars)
  - First 250 characters are most important — they appear in Google Search results, so lead with the strongest value proposition
  - Naturally weave in the provided target keywords — do NOT stuff or list them
  - Reference target cities/service areas naturally where appropriate
  - Write in third person (the business, not "we")
  - No phone numbers, URLs, or promotional language (Google policy)
  - No ALL CAPS words
  - Focus on: what the business does, what makes it unique, the areas it serves
- User message should include business name, category, address, keywords list, and cities list
- Return the `description` string from parsed output

**2. Create src/lib/google-business-info.ts:**

This file handles GBP business information reads and writes via the mybusinessbusinessinformation v1 API. Separate from google-posts.ts (which uses the v4 localPosts API).

Export `fetchCurrentDescription(params: { googleAccountId: string; locationName: string }): Promise<string | null>`:
- Create OAuth client via `createGoogleClient(googleAccountId)`
- GET `https://mybusinessbusinessinformation.googleapis.com/v1/${locationName}?readMask=profile`
- Extract and return `response.data.profile?.description ?? null`
- Wrap in try/catch — return null on error (profile may have no description)

Export `pushDescriptionToGBP(params: { googleAccountId: string; locationName: string; description: string }): Promise<{ success: boolean; error?: string }>`:
- Create OAuth client via `createGoogleClient(googleAccountId)`
- PATCH `https://mybusinessbusinessinformation.googleapis.com/v1/${locationName}?updateMask=profile.description`
- Body: `{ profile: { description } }`
- On success: return `{ success: true }`
- On error: catch, extract error message, return `{ success: false, error: message }`
- Do NOT throw — the caller needs to handle failures gracefully (save locally even if push fails)

Note: `locationName` is stored on the Profile model as e.g. "locations/456". The mybusinessbusinessinformation v1 API accepts this directly as the resource path.

**1. Create src/app/api/onboarding/description/route.ts:**

GET endpoint — fetch existing description and current GBP description:
- Auth check (getServerSession pattern)
- Read `profileId` from searchParams, return 400 if missing
- Fetch profile with googleAccountId and locationName: `prisma.profile.findUnique({ where: { id: profileId }, select: { id: true, googleAccountId: true, locationName: true } })`
- Return 404 if profile not found
- Fetch in parallel:
  1. `prisma.profileDescription.findFirst({ where: { profileId }, orderBy: { updatedAt: 'desc' } })` — latest saved description
  2. `fetchCurrentDescription({ googleAccountId: profile.googleAccountId, locationName: profile.locationName })` — live GBP description
- Return `{ description: descriptionRecord | null, currentGBPDescription: liveDescription | null }`

POST endpoint — save/update description:
- Auth check
- Accept `{ profileId: string, content: string, isApproved?: boolean }` in body
- Validate: content must be non-empty string, content.length <= 750 (return 400 if over)
- Upsert: check if a description already exists for this profileId
  - If exists: `prisma.profileDescription.update({ where: { id: existing.id }, data: { content, isApproved: isApproved ?? false, isPushed: false, pushedAt: null } })`
  - If not: `prisma.profileDescription.create({ data: { profileId, content, isApproved: isApproved ?? false } })`
- Note: when content changes, reset isPushed to false and pushedAt to null (it's a new version)
- Return the saved description record

**2. Create src/app/api/onboarding/description/generate/route.ts:**

POST endpoint — generate AI description:
- Auth check
- Accept `{ profileId: string }` in body
- Fetch profile from DB: `prisma.profile.findUnique({ where: { id: profileId } })`
- Return 404 if not found
- Fetch keywords and cities in parallel:
  ```typescript
  const [keywordRecords, cityRecords] = await Promise.all([
    prisma.profileKeyword.findMany({ where: { profileId }, orderBy: { sortOrder: 'asc' } }),
    prisma.profileCity.findMany({ where: { profileId }, orderBy: { sortOrder: 'asc' } }),
  ]);
  ```
- Call `generateDescription({ name: profile.name, category: profile.category, address: profile.address, keywords: keywordRecords.map(k => k.keyword), cities: cityRecords.map(c => c.city) })`
- Return `{ description: result }` with 200
- Wrap in try/catch, return 500 with `{ error: "Failed to generate description" }` on AI failure

**3. Create src/app/api/onboarding/description/push/route.ts:**

POST endpoint — approve and push description to GBP:
- Auth check
- Accept `{ profileId: string, content: string }` in body
- Validate content.length <= 750
- Fetch profile: `prisma.profile.findUnique({ where: { id: profileId }, select: { id: true, googleAccountId: true, locationName: true } })`
- Return 404 if not found
- Step 1: Save/upsert to DB with isApproved: true (same upsert logic as POST route above, but always set isApproved to true)
- Step 2: Push to GBP: `pushDescriptionToGBP({ googleAccountId: profile.googleAccountId, locationName: profile.locationName, description: content })`
- If push succeeds: update DB record with `isPushed: true, pushedAt: new Date()`
- Return `{ success: true, description: updatedRecord }`
- If push fails: description is STILL saved locally (isApproved: true, isPushed: false)
- Return `{ success: false, error: pushResult.error, description: savedRecord }` with status 200 (not 500 — the save succeeded, only push failed)
