<task type="auto">
  <name>Task 1: Description step component</name>
  <files>src/components/onboarding/steps/description-step.tsx</files>
  <action>
Create a "use client" component: `DescriptionStep({ profileId, onComplete }: { profileId: string; onComplete: () => Promise<void> })`.

**State:**
- `currentGBPDescription: string | null` — the live description from Google
- `aiDescription: string` — the editable AI-generated description (textarea value)
- `savedDescription: { id: string; content: string; isApproved: boolean; isPushed: boolean; pushedAt: string | null } | null` — the DB record
- `keywords: string[]` — stored keywords for usage indicators
- `loading: boolean` — initial data fetch
- `generating: boolean` — AI generation in progress
- `pushing: boolean` — approve & push in progress
- `pushSuccess: boolean` — shows success banner
- `pushError: string | null` — shows error banner
- `hasGenerated: boolean` — tracks whether we've auto-generated yet (prevents re-trigger on re-render)

**On mount (useEffect):**
1. Fetch GET /api/onboarding/description?profileId={profileId} and GET /api/onboarding/keywords?profileId={profileId} in parallel
2. Populate state from responses:
   - `currentGBPDescription` from response
   - `savedDescription` from response (may be null)
   - `keywords` from keyword response (map to string array)
   - If savedDescription exists: set `aiDescription` to savedDescription.content
3. After data loads, if NO savedDescription exists (first visit), auto-trigger generation:
   - Set `generating: true`
   - POST /api/onboarding/description/generate with profileId
   - Set `aiDescription` to response.description
   - Set `generating: false`
   - Set `hasGenerated: true`

**Layout — stacked vertically:**

**Section 1: Current GBP Description (read-only)**
- Heading: "Current Description" with subtext "Currently on your Google Business Profile"
- If `currentGBPDescription` is not null/empty: show it in a muted div (bg-gray-50 rounded-lg p-4 text-gray-600, italic)
- If null/empty: show "No description currently set on your Google Business Profile" in muted text

**Section 2: AI-Recommended Description (editable)**
- Heading: "Recommended Description" with "Regenerate" button (Sparkles icon, secondary/outline style). Disabled while generating, shows Loader2 spinner. On click: POST /api/onboarding/description/generate, update aiDescription with response.
- If generating and no aiDescription yet: show a loading skeleton (animate-pulse bg-gray-200 rounded blocks)
- Editable textarea:
  - Value bound to `aiDescription`
  - Rows: 6
  - Styling: w-full border border-gray-300 rounded-md p-3 text-sm focus:ring-2 focus:ring-blue-500 focus:border-blue-500
  - Disabled while generating or pushing
  - If character count > 750: add red border (border-red-500)

**Character counter below textarea:**
- Format: "X / 750 characters"
- Color logic: green (text-green-600) when <= 600, yellow (text-yellow-600) when 601-700, red (text-red-600) when > 700
- If over 750: show additional red text "Description exceeds 750 character limit"

**First-250 indicator:**
- Below the character counter, show: "First 250 characters visible in Google Search" with an info icon (Info from lucide-react, text-gray-400)
- Optionally highlight or mark the first 250 chars in the textarea (discretionary — a simple text note is sufficient)

**Keyword usage indicators:**
- Below the textarea area, show section: "Keyword Coverage"
- Display each stored keyword as a small chip/tag
- If the keyword appears in `aiDescription` (case-insensitive substring match): show with a green checkmark (CheckCircle2 icon, text-green-600) and bg-green-50 styling
- If NOT found: show with gray styling (bg-gray-100 text-gray-500)
- This updates live as user edits the textarea

**Action buttons area:**

If `savedDescription?.isPushed` is true (return visit, already pushed):
- Show a green "Pushed to Google" indicator with timestamp (CheckCircle2 icon, "Pushed to Google on {formatted date}")
- Show "Regenerate" button to create a new description
- Show "Continue" button (outline style) that calls `onComplete()`

If NOT pushed (first visit or previous failure):
- "Approve & Push to Google" button (bg-green-600 text-white, prominent). Disabled if:
  - aiDescription is empty
  - aiDescription.length > 750
  - pushing is true
  - Shows Loader2 spinner + "Pushing to Google..." text while pushing
- On click:
  1. Set `pushing: true`, clear `pushError`
  2. POST /api/onboarding/description/push with { profileId, content: aiDescription }
  3. If response.success:
     - Set `pushSuccess: true`, update `savedDescription` from response
     - After 2500ms (setTimeout): call `onComplete()` to auto-advance
  4. If !response.success:
     - Set `pushError: response.error || "Failed to push description to Google"`
     - `savedDescription` is still updated from response (it was saved locally)
  5. Set `pushing: false`

- "Skip for Now" link/button (text-gray-500 underline, positioned after the main button). On click:
  - If aiDescription has content: save it locally first (POST /api/onboarding/description with { profileId, content: aiDescription })
  - Then call `onComplete()`

**Success banner (when pushSuccess is true):**
- Green banner at the top of the section: bg-green-50 border border-green-200 rounded-lg p-4
- CheckCircle2 icon + "Description successfully pushed to Google Business Profile"
- "Advancing to next step..." subtext

**Error banner (when pushError is not null):**
- Red banner: bg-red-50 border border-red-200 rounded-lg p-4
- AlertCircle icon + pushError message
- "Retry" button (bg-red-600 text-white) that retries the push (same logic as Approve & Push)

**Styling:**
- Follow project conventions from keywords-cities-step.tsx
- Primary action: bg-green-600 for the approve/push button (green emphasizes "go live")
- Card sections: bg-white rounded-lg shadow-sm border border-gray-200 p-5
- Icons from lucide-react: Sparkles, Loader2, CheckCircle2, AlertCircle, Info, RotateCcw (for Regenerate)
  </action>
