    **3. Create src/components/onboarding/wizard-shell.tsx (client component):**

    The main wizard container that manages step state, navigation, and DB persistence.

    Props:
    ```typescript
    interface WizardShellProps {
      profileId: string;
      profileName: string;
      initialProgress: {
        currentStep: number;
        completedSteps: number[];
        isComplete: boolean;
      } | null;
    }
    ```

    State management (useState, NOT Redux/Zustand):
    - `currentStep: number` - initialized from initialProgress?.currentStep or 0
    - `completedSteps: number[]` - initialized from initialProgress?.completedSteps or []
    - `saving: boolean` - loading state for DB persistence

    Step navigation functions:
    - `goToStep(step: number)`: Only allow if step is in completedSteps or step === currentStep. Set currentStep, persist to DB.
    - `completeCurrentStep()`: Add currentStep to completedSteps (if not already), advance currentStep by 1, persist to DB. If this was the last step (step 6), also set isComplete = true.
    - `goBack()`: If currentStep > 0, set currentStep to currentStep - 1. Persist to DB.

    DB persistence function `persistProgress(step, completed, isComplete)`:
    - Set saving = true
    - PATCH /api/onboarding/progress with { profileId, currentStep: step, completedSteps: completed, isComplete }
    - Set saving = false
    - No optimistic UI needed -- just fire and forget with error logging to console

    Layout:
    - Top: Profile name (text-lg font-semibold text-gray-900) with "Saving..." indicator (text-xs text-gray-400) when saving
    - StepIndicator component below profile name
    - Content area (min-h-[400px] bg-white rounded-lg shadow-sm border border-gray-200 p-6 mt-6):
      - Render placeholder content for each step. Each placeholder is a centered div with:
        - Step icon (from lucide-react: Search for Keywords, MapPin for Cities, FileText for Description, Wrench for Services, SlidersHorizontal for Attributes, Settings for Settings, CheckCircle2 for Review)
        - Step name (text-lg font-medium text-gray-900)
        - "Coming in Phase {N}" subtitle (text-sm text-gray-500)
        - Step 0 (Select Profile): show "Profile selected: {profileName}" with a green checkmark as this step is auto-completed
    - Bottom navigation bar (flex justify-between mt-6):
      - Left: "Back" button (border border-gray-300 text-gray-700 hover:bg-gray-50 rounded-md px-4 py-2) -- hidden on step 0
      - Right: "Continue" button (bg-blue-600 text-white hover:bg-blue-700 rounded-md px-4 py-2) -- on last step shows "Complete Onboarding"
      - When isComplete is true after completing last step, redirect to /dashboard/onboarding with router.push and show a success message

    On mount: If initialProgress is null (first time), POST /api/onboarding/progress to initialize, then auto-complete step 0 (profile selection is done by arriving at this page).
