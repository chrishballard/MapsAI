  <action>
    Add 5 new models to prisma/schema.prisma after the existing Report model. Also add new relation fields to the Profile model.

    **New models to add:**

    1. **OnboardingProgress** - Tracks wizard state per profile:
    ```prisma
    model OnboardingProgress {
      id              String   @id @default(cuid())
      profileId       String   @unique
      profile         Profile  @relation(fields: [profileId], references: [id], onDelete: Cascade)
      currentStep     Int      @default(0)
      completedSteps  Int[]    @default([])
      isComplete      Boolean  @default(false)
      completedAt     DateTime?
      createdAt       DateTime @default(now())
      updatedAt       DateTime @updatedAt
    }
    ```

    2. **ProfileKeyword** - Target SEO keywords per profile:
    ```prisma
    model ProfileKeyword {
      id        String   @id @default(cuid())
      profileId String
      profile   Profile  @relation(fields: [profileId], references: [id], onDelete: Cascade)
      keyword   String
      sortOrder Int      @default(0)
      createdAt DateTime @default(now())
      updatedAt DateTime @updatedAt

      @@unique([profileId, keyword])
    }
    ```

    3. **ProfileCity** - Target cities/service areas per profile:
    ```prisma
    model ProfileCity {
      id        String   @id @default(cuid())
      profileId String
      profile   Profile  @relation(fields: [profileId], references: [id], onDelete: Cascade)
      city      String
      sortOrder Int      @default(0)
      createdAt DateTime @default(now())
      updatedAt DateTime @updatedAt

      @@unique([profileId, city])
    }
    ```

    4. **ProfileDescription** - AI-generated business descriptions:
    ```prisma
    model ProfileDescription {
      id          String    @id @default(cuid())
      profileId   String
      profile     Profile   @relation(fields: [profileId], references: [id], onDelete: Cascade)
      content     String    @db.Text
      isApproved  Boolean   @default(false)
      isPushed    Boolean   @default(false)
      pushedAt    DateTime?
      createdAt   DateTime  @default(now())
      updatedAt   DateTime  @updatedAt
    }
    ```

    5. **ProfileService** - AI-optimized service descriptions:
    ```prisma
    model ProfileService {
      id              String    @id @default(cuid())
      profileId       String
      profile         Profile   @relation(fields: [profileId], references: [id], onDelete: Cascade)
      serviceName     String
      description     String?   @db.Text
      isStructured    Boolean   @default(true)
      isApproved      Boolean   @default(false)
      isPushed        Boolean   @default(false)
      pushedAt        DateTime?
      createdAt       DateTime  @default(now())
      updatedAt       DateTime  @updatedAt

      @@unique([profileId, serviceName])
    }
    ```

  <action>
    **1. Create src/app/api/onboarding/progress/route.ts:**

    Follow the existing API route pattern (getServerSession check, Prisma queries, NextResponse.json).

    **GET** - Fetch onboarding progress for a profile:
    - Read `profileId` from URL searchParams
    - If no profileId, return 400: `{ error: "profileId is required" }`
    - Query `prisma.onboardingProgress.findUnique({ where: { profileId } })`
    - Return `{ progress: result }` (result may be null if no progress exists yet)

    **POST** - Create/initialize onboarding progress:
    - Read JSON body: `{ profileId: string }`
    - Validate profileId exists in Profile table
    - Use `prisma.onboardingProgress.upsert` with:
      - where: `{ profileId }`
      - create: `{ profileId, currentStep: 0, completedSteps: [], isComplete: false }`
      - update: `{}` (no-op if already exists, don't reset progress)
    - Return 201: `{ progress: result }`

    **PATCH** - Update onboarding progress:
    - Read JSON body: `{ profileId: string, currentStep?: number, completedSteps?: number[], isComplete?: boolean }`
    - Update `prisma.onboardingProgress.update({ where: { profileId }, data: { ...fields, completedAt: isComplete ? new Date() : undefined } })`
    - Return: `{ progress: result }`

    **2. Create src/app/api/onboarding/profiles/route.ts:**

    **GET** - List profiles that have NOT completed onboarding:
    - Query profiles where they either have no OnboardingProgress record OR their OnboardingProgress.isComplete is false
    - Use:
      ```typescript
      const profiles = await prisma.profile.findMany({
        where: {
          isConnected: true,
          OR: [
            { onboardingProgress: null },
            { onboardingProgress: { isComplete: false } },
          ],
        },
        select: {
          id: true,
          name: true,
          address: true,
          category: true,
          onboardingProgress: {
            select: {
              currentStep: true,
              completedSteps: true,
            },
          },
        },
        orderBy: { name: "asc" },
      });
      ```
    - Return: `{ profiles }`
    - This lets the UI distinguish between "never started" (no progress) and "in progress" (has progress but not complete)
