<task type="auto">
  <name>Task 1: Services step component</name>
  <files>src/components/onboarding/steps/services-step.tsx</files>
  <action>
Create a "use client" component: `ServicesStep({ profileId, onComplete }: { profileId: string; onComplete: () => Promise<void> })`.

**Types (local to file):**
```typescript
interface ServiceItem {
  id?: string;
  serviceName: string;
  description: string;
  isStructured: boolean;
  isApproved: boolean;
  isPushed: boolean;
  pushedAt: string | null;
}
```

**State:**
- `services: ServiceItem[]` -- the list of services with descriptions (editable, approvable)
- `availableServices: { serviceTypeId: string; displayName: string }[]` -- structured services from GBP (for initial selection)
- `loading: boolean` -- initial data fetch
- `generating: boolean` -- AI generation in progress
- `pushing: boolean` -- push in progress
- `pushSuccess: boolean` -- shows success banner
- `pushError: string | null` -- shows error banner
- `customServiceInput: string` -- text input for adding custom services
- `showSelection: boolean` -- whether service selection checklist is shown (pre-generation phase)

**On mount (useEffect):**
1. Fetch GET /api/onboarding/services?profileId={profileId}
2. Populate state from response:
   - `services` from response.services (saved services, may be empty)
   - `availableServices` from response.availableServices
3. After data loads:
   - If services array is NOT empty (return visit): set `showSelection: false`, display the service cards
   - If services array IS empty (first visit): set `showSelection: true`, display the service selection checklist

**Phase 1 UI: Service Selection (showSelection === true)**

Display a checklist of available structured services from GBP:
- Heading: "Available Services" with subtext "Select the services your business offers. We'll generate AI-optimized descriptions for each."
- Each available service as a checkbox row:
  - Checkbox (pre-checked by default) + service display name
  - User can uncheck services they don't offer
- If availableServices is empty: show "No structured services found for this business category. You can add custom services below."

**Custom service input:**
- Below the checklist: text input + "Add Service" button
- Label: "Add a custom service" with subtext "For niche services not in the list above"
- On add: append to a local `selectedCustomServices: string[]` state
- Show added custom services as removable chips below the input
- Prevent duplicates (case-insensitive check against both available services and custom list)

**"Generate Descriptions" button:**
- Prominent button below the selection area: bg-blue-600 text-white
- Text: "Generate Descriptions" with Sparkles icon
- Disabled if no services are selected (all unchecked + no custom)
- On click:
  1. Collect selected service names: checked available services (by displayName) + custom services
  2. Set `generating: true`
  3. POST /api/onboarding/services/generate with { profileId, serviceNames }
  4. Map response to ServiceItem objects (isStructured based on whether from available or custom, isApproved: false, isPushed: false)
  5. Save to DB: POST /api/onboarding/services with the full list
  6. Set `services` state, set `showSelection: false`, set `generating: false`

**Loading state during generation:**
- Show skeleton cards (animate-pulse) with text "Generating descriptions for X services..."

**Phase 2 UI: Service Cards (showSelection === false)**

Scrollable list of service cards. Each card contains:

**Service card layout:**
- Card container: bg-white border border-gray-200 rounded-lg p-4 mb-3
- If approved: left border accent (border-l-4 border-green-500)
- If pushed: left border accent (border-l-4 border-blue-500)

- **Top row:** Service name (font-medium text-gray-900) + badge showing "Structured" (bg-blue-100 text-blue-700 text-xs) or "Custom" (bg-gray-100 text-gray-600 text-xs)
- **Middle:** Editable textarea for description
  - Rows: 3
  - Styling: w-full border border-gray-300 rounded-md p-2 text-sm
  - Value bound to that service's description
  - Disabled if isPushed is true
  - On change: update the services array in state (no auto-save, saved on approve/push)
- **Character count:** Below textarea, small text "X characters" (text-gray-400)
- **Bottom row:** Action buttons
  - If NOT approved and NOT pushed:
    - "Approve" button (bg-green-50 text-green-700 border border-green-200 rounded px-3 py-1 text-sm) with CheckCircle2 icon
    - On click: set isApproved = true for this service in local state
  - If approved and NOT pushed:
    - "Approved" indicator (text-green-600 text-sm) with CheckCircle2 icon
    - "Undo" link (text-gray-400 text-xs underline) to set isApproved back to false
  - If pushed:
    - "Pushed to Google" indicator with timestamp (text-blue-600 text-sm, CheckCircle2 icon)
    - Format: "Pushed on {date}"

**Running counter above the card list:**
- Format: "X of Y services approved" (text-sm text-gray-600)
- Updates live as user approves/rejects

**Below all cards (sticky or after scrolling past all):**

**"Approve All" button:**
- Only visible if there are unapproved services remaining
- Text: "Approve All Remaining" (bg-green-50 text-green-700 border border-green-200 rounded-md px-4 py-2)
- On click: set isApproved = true for ALL services in local state

**"Push All to Google" button:**
- Prominent: bg-green-600 text-white rounded-md font-medium px-6 py-2.5
- Disabled if:
  - No services are approved
  - pushing is true
- Shows Loader2 spinner + "Pushing to Google..." text while pushing
- On click:
  1. First, save all services to DB: POST /api/onboarding/services with full services list (to persist any description edits and approval states)
  2. Set `pushing: true`, clear `pushError`
  3. POST /api/onboarding/services/push with { profileId }
  4. If response.success:
     - Set `pushSuccess: true`
     - Update all approved services in local state: isPushed = true, pushedAt = now
     - After 2500ms (setTimeout): call `onComplete()` to auto-advance
  5. If !response.success:
     - Set `pushError: response.error || "Failed to push services to Google"`
  6. Set `pushing: false`

**"Regenerate All" button:**
- Secondary style: text-gray-600 border border-gray-300 rounded-md px-4 py-2
- With RotateCcw icon
- On click: set `showSelection: true` to return to the selection phase (lets user re-select and regenerate)
- Only shown if NOT all services are pushed

**"Skip for Now" link:**
- text-gray-500 underline, below the push button
- On click:
  - Save current services to DB (POST /api/onboarding/services with full list)
  - Then call `onComplete()`

**Success banner (when pushSuccess is true):**
- Green banner at top: bg-green-50 border border-green-200 rounded-lg p-4
- CheckCircle2 icon + "Services successfully pushed to Google Business Profile"
- "Advancing to next step..." subtext

**Error banner (when pushError is not null):**
- Red banner: bg-red-50 border border-red-200 rounded-lg p-4
- AlertCircle icon + pushError message
- "Retry" button (bg-red-600 text-white) that retries the push

**Styling:**
- Follow project conventions from description-step.tsx and keywords-cities-step.tsx
- Icons from lucide-react: Sparkles, Loader2, CheckCircle2, AlertCircle, RotateCcw, Plus, X, Wrench
