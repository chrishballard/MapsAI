Implementation:
- Use `anthropic.messages.parse()` with `zodOutputFormat(ServiceDescriptionSchema)` -- same pattern as keyword-generator.ts
- Model: "claude-sonnet-4-5-20250929"
- System prompt should instruct Claude to write compelling, SEO-optimized service descriptions for a Google Business Profile. Key instructions:
  - Write a unique description for each service in the provided list
  - Each description should be 200-300 characters (service descriptions are shorter than business descriptions)
  - Naturally incorporate the provided target keywords where relevant -- do NOT force every keyword into every description
  - Reference target cities/service areas naturally where appropriate
  - Write in third person
  - Each description should differentiate itself -- avoid repetitive phrasing across descriptions
  - Focus on: what the service includes, why customers choose this business for it, and what makes their approach unique
  - No phone numbers, URLs, or promotional language (Google policy)
  - No ALL CAPS words
- User message should include business name, category, address, keywords list, cities list, and the array of service names to describe
- `max_tokens: 4096` (larger than description generator since we're generating multiple descriptions)
- Return the `services` array from parsed output
- Validate that every input serviceName has a corresponding output (log warning if any are missing but don't throw)

**Extend src/lib/google-business-info.ts with three new exported functions.**

Read the existing file first -- it already has `fetchCurrentDescription` and `pushDescriptionToGBP`. Add the following functions below the existing ones.

**1. Export `fetchStructuredServices(params: { googleAccountId: string; locationName: string }): Promise<{ serviceTypeId: string; displayName: string }[]>`:**
- Create OAuth client via `createGoogleClient(googleAccountId)`
- GET `https://mybusinessbusinessinformation.googleapis.com/v1/${locationName}?readMask=serviceItems`
- Extract `response.data.serviceItems` (may be undefined/empty array)
- Filter to only structured service items (those with `structuredServiceItem` property)
- Map to `{ serviceTypeId: item.structuredServiceItem.serviceTypeId, displayName: item.structuredServiceItem.serviceTypeId }` -- note: the display name may need to be derived from the serviceTypeId (replace underscores with spaces, title case). The GBP API returns serviceTypeId like "auto_oil_change"; convert to "Auto Oil Change" for display.
- Also include any items that have a `structuredServiceItem` but might already have a description (these are services already on the profile)
- Wrap in try/catch -- return empty array on error (profile may have no services yet)

**2. Export `fetchCurrentServices(params: { googleAccountId: string; locationName: string }): Promise<{ serviceItems: unknown[] }>`:**
- Create OAuth client via `createGoogleClient(googleAccountId)`
- GET `https://mybusinessbusinessinformation.googleapis.com/v1/${locationName}?readMask=serviceItems`
- Return the raw `{ serviceItems: response.data.serviceItems || [] }` -- the full unmodified list is needed for the merge step
- Wrap in try/catch -- return `{ serviceItems: [] }` on error

**3. Export `pushServicesToGBP(params: { googleAccountId: string; locationName: string; serviceItems: unknown[] }): Promise<{ success: boolean; error?: string }>`:**
- Create OAuth client via `createGoogleClient(googleAccountId)`
- PATCH `https://mybusinessbusinessinformation.googleapis.com/v1/${locationName}?updateMask=serviceItems`
- Body: `{ serviceItems }`
- On success: return `{ success: true }`
- On error: catch, extract error message, return `{ success: false, error: message }`
- Do NOT throw -- the caller handles failures gracefully

Note: The `serviceItems` parameter is the FULL merged list (existing + optimized). The caller (push API route) is responsible for the fetch-merge logic.

**1. Create src/app/api/onboarding/services/route.ts:**

GET endpoint -- fetch saved services and available structured services from GBP:
- Auth check (getServerSession pattern)
- Read `profileId` from searchParams, return 400 if missing
- Fetch profile: `prisma.profile.findUnique({ where: { id: profileId }, select: { id: true, googleAccountId: true, locationName: true } })`
- Return 404 if not found
- Fetch in parallel:
  1. `prisma.profileService.findMany({ where: { profileId }, orderBy: { createdAt: 'asc' } })` -- saved services
  2. `fetchStructuredServices({ googleAccountId: profile.googleAccountId, locationName: profile.locationName })` -- available services from GBP
- Return `{ services: savedServices, availableServices: structuredServices }`

POST endpoint -- save services (batch upsert):
- Auth check
- Accept `{ profileId: string, services: { serviceName: string, description?: string, isStructured: boolean, isApproved?: boolean }[] }` in body
- Validate: services must be an array, each entry must have serviceName
- Use `prisma.$transaction()` for atomicity:
  - For each service in the input array, upsert using the `@@unique([profileId, serviceName])` constraint:
    ```typescript
    prisma.profileService.upsert({
      where: { profileId_serviceName: { profileId, serviceName: service.serviceName } },
      create: { profileId, serviceName: service.serviceName, description: service.description || null, isStructured: service.isStructured, isApproved: service.isApproved ?? false },
      update: { description: service.description || null, isStructured: service.isStructured, isApproved: service.isApproved ?? false, isPushed: false, pushedAt: null },
    })
    ```
  - Note: on update, reset isPushed to false (content changed)
- Return all services for this profileId after upsert

**2. Create src/app/api/onboarding/services/generate/route.ts:**

POST endpoint -- batch generate AI service descriptions:
- Auth check
- Accept `{ profileId: string, serviceNames: string[] }` in body
- Validate: serviceNames must be a non-empty array, max 20 items
- Fetch profile: `prisma.profile.findUnique({ where: { id: profileId } })`
- Return 404 if not found
- Fetch keywords and cities in parallel:
  ```typescript
  const [keywordRecords, cityRecords] = await Promise.all([
    prisma.profileKeyword.findMany({ where: { profileId }, orderBy: { sortOrder: 'asc' } }),
    prisma.profileCity.findMany({ where: { profileId }, orderBy: { sortOrder: 'asc' } }),
  ]);
  ```
- Call `generateServiceDescriptions({ businessName: profile.name, category: profile.category, address: profile.address, keywords: keywordRecords.map(k => k.keyword), cities: cityRecords.map(c => c.city), serviceNames })`
- Return `{ services: result }` with 200
- Wrap in try/catch, return 500 with `{ error: "Failed to generate service descriptions" }` on AI failure

**3. Create src/app/api/onboarding/services/push/route.ts:**

POST endpoint -- fetch-merge-push approved services to GBP:
- Auth check
- Accept `{ profileId: string }` in body
- Fetch profile: `prisma.profile.findUnique({ where: { id: profileId }, select: { id: true, googleAccountId: true, locationName: true } })`
- Return 404 if not found

- Step 1: Fetch approved services from DB:
  ```typescript
  const approvedServices = await prisma.profileService.findMany({
    where: { profileId, isApproved: true },
  });
  ```
  - If no approved services, return `{ success: false, error: "No approved services to push" }` with 400

- Step 2: Fetch current services from GBP (the full list for merge):
  ```typescript
  const currentGBP = await fetchCurrentServices({
    googleAccountId: profile.googleAccountId,
    locationName: profile.locationName,
  });
  ```

- Step 3: Store pre-push snapshot (for future rollback capability):
  - Store the snapshot as a JSON string in a simple way. Use the Profile model or a log approach. The simplest approach: log it to console with a `[SERVICE_SNAPSHOT]` prefix for now. The CONTEXT.md defers rollback UI to a future phase, so a console log of the snapshot is sufficient. If you want to be more robust, store it in a new field or separate table -- but keep it simple for now. A `console.log("[SERVICE_SNAPSHOT]", JSON.stringify({ profileId, timestamp: new Date().toISOString(), serviceItems: currentGBP.serviceItems }))` is acceptable.

- Step 4: Merge logic:
  - Start with the current GBP service items as the base
  - For each approved service from DB:
    - If `isStructured === true`: find existing structured service item with matching serviceTypeId in the base list. If found, update its description. If not found, add a new structured service item: `{ structuredServiceItem: { serviceTypeId: serviceName, description } }`
    - If `isStructured === false` (custom/free-form): add as a free-form service item: `{ freeFormServiceItem: { category: profile.category || "General", label: { displayName: serviceName, description } } }`
  - The merge preserves any GBP services that are NOT in our approved list (no data loss)

- Step 5: Push merged list to GBP:
  ```typescript
  const pushResult = await pushServicesToGBP({
    googleAccountId: profile.googleAccountId,
    locationName: profile.locationName,
    serviceItems: mergedServiceItems,
  });
  ```

- Step 6: Update DB records on success:
  - If push succeeds: update all approved services with `isPushed: true, pushedAt: new Date()`
  - Return `{ success: true, pushedCount: approvedServices.length }`
  - If push fails: services remain saved locally with isApproved: true, isPushed: false
  - Return `{ success: false, error: pushResult.error }` with status 200 (local save succeeded)
