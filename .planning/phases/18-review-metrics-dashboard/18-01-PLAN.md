    Create `src/lib/review-metrics.ts` with these exported types and functions:

    **Types:**
    ```typescript
    export interface TrendResult { current: number; prior: number; pct: number | null }
    export interface RatingPoint { star: number; count: number }
    export interface MonthlyDataPoint { month: string; sortKey: string; volume: number; avgRating: number }
    export interface RecencyStatus { status: "good" | "warning" | "critical"; color: string; message: string }
    ```

    **Functions (all pure, no side effects):**

    1. `computeTrend(reviews: Array<{ reviewDate: Date }>, now: Date): TrendResult` — per D-08, rolling 30-day window. `cutoff = now - 30 days`, `priorCutoff = now - 60 days`. current = reviews with reviewDate >= cutoff. prior = reviews with reviewDate >= priorCutoff AND < cutoff. pct = prior === 0 ? null : Math.round(((current - prior) / prior) * 100).

    2. `computeRatingDistribution(reviews: Array<{ rating: number }>): RatingPoint[]` — Group by rating 1-5. Return array ordered [5,4,3,2,1] (top-to-bottom for horizontal bar chart per D-05). Initialize all 5 buckets to 0 before counting.

    3. `computeMonthlyData(reviews: Array<{ reviewDate: Date; rating: number }>, now: Date): MonthlyDataPoint[]` — Build 12-month bucket map using UTC. For each month i from 11 down to 0: key = `YYYY-MM` from `Date.UTC(now.getUTCFullYear(), now.getUTCMonth() - i, 1)`. Iterate reviews, bucket by `getUTCFullYear()`-`getUTCMonth()+1`. Return array with `month` formatted as "Jan 2026" (using `toLocaleDateString("en-US", { month: "short", year: "numeric", timeZone: "UTC" })`), `sortKey`, `volume`, `avgRating` (rounded to 1 decimal, 0 if no reviews in month).

    4. `computeDaysSince(date: Date | null, now: Date): number | null` — Returns null if date is null. Otherwise `Math.floor((now.getTime() - date.getTime()) / (24 * 60 * 60 * 1000))`.

    5. `getRecencyStatus(daysSince: number | null): RecencyStatus` — per D-09/D-10: null => {status:"critical", color:"red", message:"No reviews yet"}. <=14 => {status:"good", color:"green", message:"Profile is active"}. <=30 => {status:"warning", color:"amber", message:"Consider requesting reviews"}. >30 => {status:"critical", color:"red", message:"No reviews in over a month"}.

    6. `formatDataThrough(date: Date): string` — per D-13: Returns `"Data through " + date.toLocaleDateString("en-US", { month: "short", day: "numeric", year: "numeric", timeZone: "UTC" })`.

    Create `tests/lib/review-metrics.test.ts` following the optimization-score.test.ts pattern:
    - Import from `@/lib/review-metrics`
    - Use `const now = new Date()` and `const daysAgo = (d: number) => new Date(now.getTime() - d * 24 * 60 * 60 * 1000)` helpers
    - Test each function with the behaviors listed above
    - At least 15 test cases total covering: normal cases, edge cases (empty arrays, null dates, zero prior period), boundary values (14 vs 15 days, 30 vs 31 days)

    Write tests FIRST (RED), then implement functions (GREEN).
