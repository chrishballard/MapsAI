## Phase 14 UI Scope

Phase 14 has **no persistent user-facing UI.** It ships three artifacts:

1. `src/lib/optimization-score.ts` — pure TypeScript function, no rendering
2. `src/components/ui/chart.tsx` — shadcn chart primitive installed via CLI, no new visual design decisions required
3. `src/app/dashboard/_dev/chart-smoke/page.tsx` — temporary dev-only smoke test page, deleted after `npm run build` succeeds

The design contract below covers: (a) the smoke test page layout, (b) the score badge color semantics established in this phase for downstream consumption by Phases 15, 16, and 17.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#fafafa` (`--background`, zinc-50) | Page backgrounds, main surfaces |
| Secondary (30%) | `#FFFFFF` (`--card`) + `#f4f4f5` (`--secondary`) | Cards, sidebar, nav |
| Accent (10%) | `#7c3aed` (`--primary`, brand-600) | See reserved list below |
| Destructive | `oklch(0.58 0.22 27)` (`--destructive`) | Destructive actions only — none in Phase 14 |

Accent (`#7c3aed`) reserved for:
- Smoke test page: no accent usage (minimal dev page)
- Score badge "green" state: NOT violet — use `emerald` semantic (see Score Color Contract below)
- Score badge "amber" state: NOT violet — use `amber` semantic (see Score Color Contract below)
- Score badge "red" state: NOT violet — use `red` semantic (see Score Color Contract below)

Violet accent is reserved for: primary action buttons, focus rings, active nav items, sidebar primary.

Source: `globals.css` `:root` block (confirmed), brand-guidelines.md (confirmed).

## Smoke Test Page Layout Contract

Route: `src/app/dashboard/_dev/chart-smoke/page.tsx`
Lifecycle: Create for build validation. Delete `_dev/` directory after `npm run build` exits 0.
Authentication: Protected by existing dashboard layout auth — no additional gate needed.

Layout:
- Outer: `'use client'` directive at top of file
- Container: `<div className="p-8 space-y-8">`
- Heading: `<h1 className="text-xl font-semibold text-foreground">Chart Smoke Test</h1>`
- Subtext: `<p className="text-sm text-muted-foreground">Dev validation page — delete after build passes.</p>`
- Chart section: `<ChartContainer config={...} className="min-h-[200px]">` with one `LineChart`
- QR section: `<QRCodeSVG value="https://rankmaps.io" size={128} />`

ChartContainer MUST have `className="min-h-[200px]"` — prevents hydration mismatch from zero-height ResponsiveContainer. Source: RESEARCH.md Pitfall 1.

Chart color reference: use `var(--chart-1)` (not `hsl(var(--chart-1))`). Source: RESEARCH.md Pitfall 3 — `globals.css` defines `--chart-1: #7c3aed` as hex.
