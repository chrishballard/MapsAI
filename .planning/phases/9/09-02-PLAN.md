<task type="auto">
  <name>Task 1: Keywords & Cities step component</name>
  <files>src/components/onboarding/steps/keywords-cities-step.tsx</files>
  <action>
Create a "use client" component: `KeywordsCitiesStep({ profileId, onComplete }: { profileId: string; onComplete: () => Promise<void> })`.

**State:**
- `keywords: string[]` — the editable keyword list
- `cities: string[]` — the editable city list
- `suggestions: { keyword: string; reasoning: string }[] | null` — AI suggestions (shown before user accepts)
- `generating: boolean` — loading state for AI generation
- `saving: boolean` — loading state for save
- `editingKeywordIndex: number | null` — index of keyword being inline-edited
- `newKeyword: string` — input for manually adding a keyword
- `newCity: string` — input for adding a city

**On mount (useEffect):**
- Fetch GET /api/onboarding/keywords?profileId={profileId} and GET /api/onboarding/cities?profileId={profileId} in parallel
- Populate `keywords` and `cities` state from response

**Layout — two sections stacked vertically inside a single card container:**

**Section 1: Keywords**
- Heading: "Target Keywords" with subtext "Up to 10 keywords that describe your business and services"
- "Generate with AI" button (Sparkles icon from lucide-react, bg-blue-600 primary style). Disabled while generating, shows Loader2 spinner.
- When clicked: POST /api/onboarding/keywords/generate with profileId
- When suggestions arrive, show them in a suggestions panel: each suggestion as a row with the keyword text, a small gray "reasoning" tooltip or subtext, and an "Add" button (Plus icon). Only show suggestions that are not already in the keywords list.
- Current keywords displayed as an ordered list of pill/chip elements. Each chip shows:
  - Drag handle (GripVertical icon, text-gray-400) — for visual affordance (actual drag reorder is discretionary, simpler: up/down arrow buttons)
  - The keyword text (click to edit inline)
  - Remove button (X icon, text-gray-400 hover:text-red-500)
- Below chips: an input field + "Add" button for manually adding a keyword. Disable the Add button if keywords.length >= 10 or input is empty.
- Show "{keywords.length}/10" counter

**Section 2: Cities / Service Areas**
- Heading: "Target Cities" with subtext "Up to 3 cities or service areas"
- City list as chips (same style as keywords but without reorder controls)
- Input field + "Add" button. Disable if cities.length >= 3 or input is empty.
- Show "{cities.length}/3" counter

**Bottom: "Save & Continue" button**
- On click: POST keywords and cities in parallel, then call `onComplete()`
- Disabled while saving, shows Loader2 spinner
- Only enabled if at least 1 keyword is set

**Styling:**
- Use project conventions: bg-white rounded-lg shadow-sm border border-gray-200 for card sections
- Primary buttons: bg-blue-600 text-white rounded-md font-medium hover:bg-blue-700
- Secondary/outline buttons: border border-gray-300 text-gray-700 hover:bg-gray-50 rounded-md
- Chip/pill style: bg-blue-50 text-blue-700 rounded-full px-3 py-1 text-sm with inline flex for icon+text+remove
- Icons from lucide-react: Sparkles, Plus, X, GripVertical, Loader2, MapPin
