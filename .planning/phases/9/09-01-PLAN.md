**1. Create src/lib/keyword-generator.ts:**

Define a Zod schema for structured AI output:
```typescript
const KeywordSuggestionsSchema = z.object({
  keywords: z.array(z.object({
    keyword: z.string(),
    reasoning: z.string(),
  })).min(5).max(10),
});
```

Export `generateKeywordSuggestions(profile: { name: string; category: string | null; address: string | null }): Promise<{ keyword: string; reasoning: string }[]>`.

Implementation:
- Use `anthropic.messages.parse()` with `zodOutputFormat(KeywordSuggestionsSchema)` — same pattern as post-generator.ts
- Model: "claude-sonnet-4-5-20250929" (same as post generator)
- System prompt should instruct Claude to generate 8-10 specific, localized SEO keywords for a Google Business Profile. Emphasize: keywords must be specific to the business type and location (not generic like "great service"), include location-qualified keywords (e.g., "dentist in Springfield" not just "dentist"), mix of service keywords and location keywords, avoid duplicating the business name as a keyword.
- User message should include business name, category, and full address.
- Return the `keywords` array from parsed output.

**2. Create src/app/api/onboarding/keywords/generate/route.ts:**

POST endpoint. Auth check using getServerSession pattern (see progress/route.ts).
- Accept `{ profileId: string }` in body
- Fetch profile from DB: `prisma.profile.findUnique({ where: { id: profileId } })`
- Return 404 if not found
- Call `generateKeywordSuggestions({ name: profile.name, category: profile.category, address: profile.address })`
- Return `{ keywords: result }` with 200
- Wrap in try/catch, return 500 on AI failure

**3. Create src/app/api/onboarding/keywords/route.ts:**

GET endpoint:
- Auth check
- Read `profileId` from searchParams
- `prisma.profileKeyword.findMany({ where: { profileId }, orderBy: { sortOrder: 'asc' } })`
- Return `{ keywords }`

POST endpoint (save/replace all keywords):
- Auth check
- Accept `{ profileId: string, keywords: string[] }` in body
- Validate: keywords.length <= 10, each keyword trimmed and non-empty
- Delete all existing keywords for profileId: `prisma.profileKeyword.deleteMany({ where: { profileId } })`
- Create new ones with sortOrder matching array index: `prisma.profileKeyword.createMany({ data: keywords.map((kw, i) => ({ profileId, keyword: kw.trim(), sortOrder: i })) })`
- Use a transaction for delete+create
- Return the new keywords list

**4. Create src/app/api/onboarding/cities/route.ts:**

GET endpoint:
- Auth check
- Read `profileId` from searchParams
- `prisma.profileCity.findMany({ where: { profileId }, orderBy: { sortOrder: 'asc' } })`
- Return `{ cities }`

POST endpoint (save/replace all cities):
- Auth check
- Accept `{ profileId: string, cities: string[] }` in body
- Validate: cities.length <= 3, each city trimmed and non-empty
- Delete all existing cities for profileId, create new ones with sortOrder
- Use a transaction
- Return the new cities list

**2. Modify src/app/api/posts/generate/route.ts:**

Inside the `for (const profileId of body.profileIds)` loop, after fetching the profile, also fetch keywords and cities:

```typescript
const [keywordRecords, cityRecords] = await Promise.all([
  prisma.profileKeyword.findMany({
    where: { profileId: profile.id },
    orderBy: { sortOrder: 'asc' },
  }),
  prisma.profileCity.findMany({
    where: { profileId: profile.id },
    orderBy: { sortOrder: 'asc' },
  }),
]);
```

Pass them to generateMonthlyPosts:
```typescript
const generated = await generateMonthlyPosts(
  {
    name: profile.name,
    category: profile.category,
    address: profile.address,
    keywords: keywordRecords.map(k => k.keyword),
    cities: cityRecords.map(c => c.city),
  },
  customPrompt ?? undefined
);
```

This is fully backward compatible — profiles with no keywords/cities will pass empty arrays which are filtered out by the `?.length` checks in post-generator.ts.
